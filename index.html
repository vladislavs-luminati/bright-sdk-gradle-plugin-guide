Bright SDK Update Plugin Integration Guide

Introduction

The Bright SDK Update Plugin is designed to automate the process of updating the Bright SDK within your project. By integrating this plugin, developers can effortlessly switch between SDK versions, ensuring they are always working with the latest features and fixes. This plugin also simplifies debugging by allowing easy transitions between different versions of the SDK.

Benefits

  • Automated Updates: Automatically fetches and installs the latest version of the Bright SDK, reducing manual overhead.
  • Seamless Integration: Simplifies the process of adding the Bright SDK as a dependency in your project’s build.gradle file.
  • Version Management: Allows easy switching between different SDK versions, which is particularly useful for testing and debugging.
  • Local Caching: Supports caching of SDK files, minimizing the need for repeated downloads.

Prerequisites

  • Java Development Kit (JDK): Version 17 or higher, with backward compatibility for Java 8.
  • Gradle: Version 6.7 or later.
  • Internet Access: Required for downloading the SDK version information and the SDK itself.
  • GitHub Credentials: To access the plugin repository on GitHub Packages, you'll need your GITHUB_USERNAME and GITHUB_TOKEN for authentication.

Getting Started

System Requirements

  1. JDK: Version 17 or higher.
  2. Gradle: Version 6.7 or later.
  3. Supported Platforms: Any platform where Java and Gradle can run (e.g., Windows, macOS, Linux).
  4. GitHub Credentials: Required for accessing private repositories or GitHub Packages.

Installation Instructions

Step-by-Step Integration into the Application

Step 1: Adding the Plugin to Your Project

  1. Create a brightsdk.gradle File in the app/ Directory:

    Add a file named brightsdk.gradle in your app/ directory with the following content:

    buildscript {
    repositories {
        maven {
            url = uri("https://maven.pkg.github.com/vladislavs-luminati/gradle-plugins")
            credentials {
                username = System.getenv("GITHUB_USER") ?: project.findProperty("gpr.user")
                password = System.getenv("GITHUB_TOKEN") ?: project.findProperty("gpr.token")
            }
        }
    }
    dependencies {
        classpath 'com.brightdata:brightsdkupdater:1.+'
    }
}

apply plugin: com.brightdata.BrightSdkPlugin

tasks.named("preBuild").configure {
    dependsOn updateBrightSdk
}

  2. Register plugin in app/build.gradle:

    Add the following lines to your app/build.gradle file:

    apply from: 'brightsdk.gradle'

Step 2: Verifying the Integration

Run your Gradle build process as usual (./gradlew build). The updateBrightSdk task will automatically execute before the preBuild task, ensuring that the latest version of the Bright SDK is used in your project.

Explanation:

  • brightsdk.gradle in app/ Directory: This file contains the configuration and application of the Bright SDK Update Plugin.
  • Auto-Update Setup: By adding the specified lines in app/build.gradle, you ensure that the SDK update runs before the preBuild task, which is an early stage in the build process.

Plugin Configuration

Basic Configuration

  • dir: The directory where the SDK files will be stored (default: "libs").
  • version: The SDK version to use (default: "latest").
  • ext: The file extension for the SDK files (default: "brightsdk").
  • useLocalCache: Whether to use a local cache for the SDK files (default: false).

Advanced Configuration

Modify these configurations directly in your build.gradle file under the plugin configuration block.

Updating the Plugin

Checking for Updates

The plugin will automatically check for the latest SDK version based on the configuration in gradle.properties.

Updating the Plugin

If a newer version is available, the plugin will download and replace the existing SDK files.

Backward Compatibility

The plugin supports backward compatibility with older SDK versions, allowing you to switch between versions as needed.

Best Practices

  • Keep Credentials Secure: Ensure that your gradle.properties file is not included in version control.
  • Use Local Cache: Enable local caching if you frequently switch between SDK versions to reduce download times.
  • Regular Updates: Regularly check for and update to the latest SDK version to benefit from the latest features and fixes.

How to Generate Your GitHub Token

To access private repositories or GitHub Packages, you need a GitHub token, also known as a Personal Access Token (PAT). Follow these steps to generate your token:

Step 1: Log in to GitHub

  1. Open your web browser and go to github.com.
  2. Log in to your GitHub account using your credentials.

Step 2: Navigate to Settings

  1. In the top-right corner of the GitHub page, click on your profile picture to open the dropdown menu.
  2. Select Settings from the dropdown menu.

Step 3: Access Developer Settings

  1. On the left sidebar, scroll down and click on Developer settings.
  2. In the Developer settings menu, select Personal access tokens.

Step 4: Generate a New Token

  1. Click on the Generate new token button.
  2. You may be prompted to enter your GitHub password again for security purposes.

Step 5: Configure Your Token

  1. Note: Give your token a descriptive name in the Note field (e.g., "Bright SDK Plugin Access").
  2. Expiration: Set an expiration date for your token based on your security preferences (e.g., 30 days, 60 days, etc.).
  3. Scopes:
    • Required Scopes:
      • repo: Full control of private repositories (necessary if you're accessing private repos).
      • write:packages: Upload packages to GitHub Package Registry.
      • read:packages: Download and install packages from GitHub Package Registry.
      • delete:packages: (Optional) Delete packages from GitHub Package Registry.
    • Optional Scopes:

      You can check other scopes based on your requirements, but for basic access to packages and repos, the above are typically sufficient.

Step 6: Generate and Save the Token

  1. Once you've selected the necessary scopes, scroll down and click the Generate token button.
  2. Important: Once the token is generated, copy it immediately. GitHub will not show the token again after you navigate away from the page.

Step 7: Registering Your Username and Token in gradle.properties

Instead of setting environment variables, you can securely store your GitHub username and token in your project's gradle.properties file. This method integrates directly with your Gradle build script.

  1. Locate the gradle.properties File:

    You can find the gradle.properties file in your project's root directory, or you can create one if it doesn’t already exist.

    Alternatively, you can place these properties in the global gradle.properties file located in the .gradle directory of your user’s home folder (~/.gradle/gradle.properties on macOS/Linux, C:\Users\YourUsername\.gradle\gradle.properties on Windows).

  2. Add Your GitHub Username and Token:

    Open the gradle.properties file in a text editor.

    Add the following lines, replacing your_github_username and your_generated_token with your actual GitHub username and the token you generated:

    gpr.user=your_github_username
gpr.token=your_generated_token
  3. Use the Properties in Your Build Script:

    Your Gradle build script will automatically use the username and token stored in gradle.properties when accessing GitHub Packages. There’s no need to modify your build script; it will read these values when needed.

  4. Security Tip:

    Ensure that your gradle.properties file is not included in version control (e.g., add it to .gitignore if necessary) to keep your credentials secure.

By storing your GitHub credentials in gradle.properties, you ensure that your build process can authenticate with GitHub Packages seamlessly while keeping your credentials out of your codebase.
